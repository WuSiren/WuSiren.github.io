从GitHub中删除commit提交历史

发表于2021-05-05|更新于2021-05-10|技术技能
|字数总计:390|阅读时长:1分钟|阅读量:

应用场景1：偶然发现GitHub中之前提交的commit中包含个人敏感信息，想将有关commit从GitHub版本历史记录中删除。
应用场景2：某篇博文在没有加密的情况下发布了，之后意识到需要加密，但是此时加密后再发布，GitHub中仍保留此前未加密的历史，需要将其删除。

这里提供一个办法，能够删除从某个历史commit之后提交的所有commit和所有尚未提交的修改。

如果是一般的代码库，可参考这个链接：https://blog.csdn.net/djj810625/article/details/101472341 。

如果是hexo博客，操作有一些不一样，步骤如下：

  1. 在hexo博客主目录下打开文件夹.deploy_git，在其中打开git bash。或者在vs code中打开此目录。
  2. 找到要回退到的历史commit节点，复制其commit id，然后使用代码git reset --hard commit_id（将commit_id替换成相应的hash值）。或者在安装了Git Graph插件的vs code中打开Git Graph，在要回退到的commit节点上右键选择Reset current branch to this Commit...，然后再选择Hard - Discard all changes->Yes, reset即可。这时会发现，该节点之后提交的commit都删除了。
  3. 然后再回到博客主目录下打开git bash，重新hexo clean && hexo g && hexo d即可。

再次提醒：该方法将删除从某个历史commit之后提交的所有commit和所有尚未提交的修改，请务必慎重使用！！！
